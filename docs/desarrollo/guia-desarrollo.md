# Guía de Desarrollo

Este documento detalla los procesos formales que vamos a seguir de ahora en adelante para agregar nuevas funcionalidades, actualizar áreas del sistema o corregir errores significativos (fixes).

## Pasos del Proceso de Desarrollo

1. **Analizar archivos pertinentes:**
   Estudiar y comprender los archivos relacionados al módulo nuevo que se va a agregar, actualizar o fixear antes de tomar acción.

2. **Crear un plan detallado:**
   Desarrollar un paso a paso detallado de todo lo que se hará, tanto a nivel lógico como a nivel estructural y de arquitectura en la aplicación.

3. **Implementación del plan:**
   Desarrollar y ejecutar de manera iterativa el plan previamente establecido.

4. **Documentación correspondiente:**
   Registrar meticulosamente todo lo realizado en la bitácora correspondiente (`bitacora-front` o `bitacora-back`), formateando como Feature o Fix, y posteriormente actualizar el archivo de la carpeta `resumen` de ser necesario.

5. **Revisión de Arquitectura:**
   Analizar si la funcionalidad agregada o modificada introdujo algún cambio en el diseño, flujo o tecnologías generales del sistema. En caso afirmativo, actualizar el documento `arquitectura-general.md` dentro de la carpeta `arquitectura/` para mantenerlo siempre como la fuente de verdad.
